# hyppo.storage

Локальное файловое хранилище объектов на основе `cloudpickle`. Класс `Database` предоставляет простой интерфейс сохранения и загрузки произвольных Python-объектов: `Database.set_root(...)` задаёт корневую директорию хранилища, `Database.save(obj, name, storage=..., description=...)` сериализует и сохраняет объект в указанном подкаталоге, а `Database.load(name, storage=...)` возвращает обёртку `Pickled` с загруженным объектом (доступным через атрибут `.obj`). Класс `Pickled` поддерживает хуки `save_data_hook`/`load_data_hook`, вызываемые до сохранения и после загрузки соответственно, а также автоматически фиксирует время сохранения в поле `when_saved`. Этот модуль служит слоем персистентности для промежуточных и итоговых артефактов виртуальных экспериментов — обученных моделей, вычисленных структур и результатов запусков, используемых репозиторием метаинформации (`hyppo.metadata_repository`) для кеширования между запусками планировщика.
