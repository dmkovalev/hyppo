# hyppo.actions

Реестр типизированных операций (действий), которые могут вызываться агентами через MCP. Декоратор `@action` (модуль `registry`) регистрирует функцию в глобальном реестре `ACTION_REGISTRY` вместе с её спецификацией `ActionSpec`: типом операции, уровнем доверия (`TrustLevel` — `safe`, `compute`, `staging`, `production`), Pydantic-схемами входа и выхода, списком допущенных ролей агентов (`AgentRole` — Coordinator, Geologist, ReservoirEngineer, ProductionEngineer, Economist, Auditor) и флагом обязательного аудита. Подмодули `diff`, `version` и `virtual_experiment` определяют конкретные действия — сравнение версий гипотез, регистрацию/получение/пометку версий (например, `register_hypothesis_version`) и операции над виртуальным экспериментом; все они импортируются в `__init__` ради побочного эффекта наполнения реестра. Взаимодействие с версиями идёт исключительно через `hyppo.versioning.version_store`, что удерживает пакет действий на верхнем уровне зависимостей и избегает циклов импорта. Реестр действий образует контролируемую поверхность API, которую MCP-сервер (`hyppo.mcp`) транслирует во внешние MCP-инструменты.

Подробная справка по сигнатурам генерируется из docstrings (на английском):

::: hyppo.actions
