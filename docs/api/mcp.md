# hyppo.mcp

Сервер Model Context Protocol (MCP) для платформы Hyppo. Функция `create_server` создаёт объект `Server("hyppo")` и регистрирует на нём инструменты (`register_tools`) и ресурсы (`register_resources`); асинхронные функции `run_stdio` и подобные ей запускают сервер на соответствующем транспорте (stdio и др.), а `cli.py` предоставляет точку входа командной строки. Модуль `tools` транслирует записи глобального реестра действий `hyppo.actions.ACTION_REGISTRY` в MCP-инструменты: каждому действию сопоставляется MCP `Tool` с аннотацией уровня доверия (`trust`) и допущенных ролей (`roles`), а вызовы протоколируются в журнал аудита `hyppo.mcp.audit`. Модуль `resources` публикует как MCP-ресурсы вспомогательные текстовые артефакты — например, промпты персон агентов (`personas/lattice_steward.md`). Таким образом `hyppo.mcp` — это транспортный и протокольный слой, позволяющий внешним LLM-агентам безопасно вызывать типизированные действия hyppo и получать доступ к решётке гипотез, планам пересчёта и персонам агентов через стандартный протокол MCP.

Подробная справка по сигнатурам генерируется из docstrings (на английском):

::: hyppo.mcp
