# hyppo.manager

Оркестратор полного жизненного цикла виртуального эксперимента (раздел 3.1.2 диссертации). Класс `Manager` связывает воедино все остальные подсистемы hyppo и проводит эксперимент через четыре стадии: (1) инициализация — создание эксперимента и сохранение его в репозитории метаинформации; (2) построение решётки гипотез из структур эксперимента (`hyppo.lattice_constructor`); (3) планирование — определение множеств P_ne/P_e с учётом кеширования (`hyppo.planner`); (4) исполнение — запуск моделей и сохранение результатов (`hyppo.runner`). Конструктор принимает путь к базе данных, порог качества R² (`r2_threshold`) и максимальное число повторных попыток (`max_retries`), после чего Manager координирует взаимодействие `MetadataRepository` и `Runner`, предоставляя единую точку входа для запуска виртуального эксперимента от конфигурации до готовых результатов.

Подробная справка по сигнатурам генерируется из docstrings (на английском):

::: hyppo.manager
