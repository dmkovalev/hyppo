# hyppo.planner

Планировщик виртуальных экспериментов, реализующий Алгоритм 4 диссертации: «Построение плана виртуального эксперимента». Принимает конфигурацию эксперимента и решётку гипотез (построенную `hyppo.lattice_constructor`) и определяет, какие гипотезы требуют пересчёта моделей (множество P_ne), а какие можно взять из кеша репозитория метаинформации (множество P_e). Ключевые сущности: датакласс `ExecutionPlan` — результат планирования с явным разбиением на P_ne и P_e; класс `Planner`, инкапсулирующий логику построения плана пересчёта; функция `build_optimal_plan`, реализующая оптимизацию по Теореме 1 диссертации, минимизирующую суммарную стоимость исполнения при заданных ограничениях на кеширование. План пересчёта, полученный планировщиком, передаётся исполнителю (`hyppo.runner`), который выполняет гипотезы из P_ne в топологическом порядке причинного упорядочивания.

Подробная справка по сигнатурам генерируется из docstrings (на английском):

::: hyppo.planner
