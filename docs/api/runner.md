# hyppo.runner

Исполнитель виртуальных экспериментов (`Runner`), описанный в разделе 3.1.7 диссертации. Выполняет модели гипотез в топологическом порядке, заданном причинным упорядочиванием решётки, с автоматическими повторными попытками при сбое (по умолчанию k=3) и каскадным распространением статуса `SKIPPED` на зависимые гипотезы, если их предшественник не был успешно вычислен. Перечисление `Status` фиксирует статус исполнения отдельной гипотезы: `SUCCESS`, `FAILED` или `SKIPPED` — это техническая характеристика запуска модели, принципиально отличная от эпистемического статуса (`hyppo.core.EpistemicStatus`), который отражает научный вердикт о гипотезе после сопоставления с данными. Датакласс `RunResult` собирает результат исполнения гипотезы: статус, метрики качества и данные, необходимые для последующего вычисления эпистемического статуса. Runner — единственное место в системе, где собираются входные данные (R², AIC, конкурирующие гипотезы) для функции `evaluate_status` и записывается итоговый эпистемический статус.

Подробная справка по сигнатурам генерируется из docstrings (на английском):

::: hyppo.runner
