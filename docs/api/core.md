# hyppo.core

Базовый пакет предметной области: OWL-онтология виртуального эксперимента (Определение 1) и функция перехода эпистемического статуса гипотезы (Раздел 2). Здесь задаётся `virtual_experiment_onto` — корневая онтология, к которой привязываются классы гипотез, моделей и экспериментов из остальных подпакетов hyppo. Класс `EpistemicStatus` перечисляет возможные научные вердикты по гипотезе: `PROPOSED` (предложена, ещё не оценена), `SUPPORTED` (метрика качества R² превышает порог θ_sup), `REFUTED` (метрика ниже порога) и `SUPERSEDED` (конкурирующая гипотеза статистически значимо лучше по ΔAIC > θ_aic). Функция `evaluate_status` — чистая функция перехода состояний, реализующая эту логику независимо от исполнения экспериментов, что позволяет тестировать её изолированно. Эпистемический статус принципиально отличается от статуса исполнения (`SUCCESS`/`FAILED`/`SKIPPED`), который отслеживается модулем `hyppo.runner`: первый — это научный вердикт, второй — факт того, отработала ли модель технически. Значения по умолчанию (θ_sup = 0.7, θ_aic = 10) соответствуют консервативным порогам Бернэма-Андерсона, принятым в диссертации.
