# hyppo.adapters

Адаптеры, соединяющие платформу виртуальных экспериментов Hyppo с внешними системами. Основной пример — адаптер к WFOpt (`wfopt_adapter`) для виртуального эксперимента по заводнению нефтяного пласта: функция `build_oil_virtual_experiment` строит OWL DL-представления шести суб-гипотез гибридной модели CRM (capacitance-resistance model) и отображает их на дискретное пространство гиперпараметров, заданное в конфигурации настройки WFOpt (см. раздел 4.10 диссертации), а `run_oil_experiment_demo` запускает демонстрационный прогон такого эксперимента. Важно, что модуль не импортирует сам пакет `wfopt` — он строит исключительно онтологические артефакты, зеркалирующие архитектуру модели WFOpt, что позволяет проводить формальный вывод (reasoning) над структурой эксперимента независимо от исполняемого кода оптимизатора. Пакет служит примером того, как доменные структуры прикладной задачи (нефтепромысловая инженерия) выражаются в терминах решётки гипотез и онтологии `hyppo.core`/`hyppo.ontology`.
