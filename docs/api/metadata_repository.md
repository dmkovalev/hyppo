# hyppo.metadata_repository

Реляционный репозиторий метаинформации о виртуальных экспериментах, описанный в разделе 3.1.8 диссертации. Класс `MetadataRepository` использует SQLite для хранения результатов вычислений с составным ключом (`hypothesis_id`, `config_hash`), что позволяет кешировать результаты по гипотезе и конкретной конфигурации входных параметров — именно на этот кеш опирается планировщик (`hyppo.planner`) при определении множества гипотез P_e, не требующих пересчёта. Датакласс `ResultRecord` описывает единицу хранения: идентификатор гипотезы, конфигурацию, метрики качества и статус исполнения (`SUCCESS`/`FAILED`/`SKIPPED`) с меткой времени. Класс `SharedCache` (модуль `shared_cache`) реализует разделяемый между несколькими экспериментами слой кеширования для сценариев, в которых разные виртуальные эксперименты используют пересекающиеся гипотезы (соответствует правилу 13 из `hyppo.ontology` — обнаружение совместного использования между экспериментами). Репозиторий — центральный компонент, обеспечивающий переиспользование ранее вычисленных фрагментов, ради которого и строится план пересчёта.
